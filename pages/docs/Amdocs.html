Configurations

Welcome to our guide on editing the configuration files for the Ascendant Mobs mod. This tutorial will help you understand how to adjust settings that control mob difficulty as you explore further into the Minecraft world.

Distance Scale Settings

                                    ["Distance Scale Settings"]
                                    # Recommended over 25,000
                                    scale_distance = 55000.0
                                    # The scale type can be 'vertical', 'horizontal', 'both', 'random', or 'MOTP'
                                    scale_type = "both"
                                    day_factor = 1.0
                                    random_min = 0.0
                                    random_max = 200.0

This section allows you to configure the distance scale for mob spawning. The scale_distance is recommended to be over 25,000 for optimal performance. The scale_type determines the method of scaling:

  • vertical: Applies scaling based on vertical distance.
  • horizontal: Applies scaling based on horizontal distance.
  • both: Applies scaling in both dimensions.
  • random: Applies random scaling.
  • MOTP: Uses MOTP - Leveling System

The day_factor affects how day/night cycles influence mob levels, while random_min and random_max set the bounds for random effects.

Mobs Levels Settings

                                    ["Mobs Levels Settings"]
                                    base_level = 0.0
                                    scale_factor = 0.5
                                    base_level_nether = 50.0
                                    base_level_end = 100.0
                                    # Mobs after a certain level get random buffs 
                                    random_effects = true
                                    #[1-200]
                                    random_effects_level = 100.0
                                    #[1-100]
                                    random_effects_chance = 10.0
                                    base_level_territory = 75.0
                                    max_level_overworld = 75.0
                                    max_level_nether = 100.0
                                    max_level_end = 200.0

This section configures the levels for mobs in different dimensions:

  • base_level: The starting level for mobs.
  • scale_factor: A multiplier for scaling mob levels.
  • base_level_nether and base_level_end: Starting levels for mobs in the Nether and the End respectively.
  • random_effects: Enables random buffs for mobs after a certain level.
  • random_effects_level: The level at which mobs can gain random effects.
  • random_effects_chance: The percentage chance for random effects to occur.
  • max_level_overworld, max_level_nether, and max_level_end: The maximum level for mobs in each dimension.

Ingame Overlay Settings

                                    ["Ingame Overlay Settings"]

Control whether the level name is displayed in the game overlay. Setting display_lvl_name to true will show the current level of mobs on the screen.

Blacklist Configuration

The list_config.toml file allows you to customize the behavior of certain mobs in your mod by excluding specific entities from the leveling system.

Configuration Details:
                                [Blacklist]
                                # Write the registry name of the entities
                                banned = [""]
                                death_messages = []
                                # Please respect this format 'xxx/namespace:mobname' where xxx is the level written in 3 digits. For example '010/minecraft:zombie'
                                locked_mobs = ["100/minecraft:ender_dragon", "150/minecraft:wither"]

The blacklist configuration includes the following options:

  • banned: This array allows you to specify entities that should be excluded from the leveling system. Simply enter the registry names of the entities you wish to ban, leaving the quotes empty to start.
  • death_messages: This array can hold custom death messages that will not be displayed when the banned mobs are defeated.
  • locked_mobs: Use this setting to lock specific mobs at a designated level. The format to follow is xxx/namespace:mobname, where xxx is the mob's level written in three digits. For example:
    • 100/minecraft:ender_dragon locks the Ender Dragon at level 100.
    • 150/minecraft:wither locks the Wither at level 150.
    This feature is useful for maintaining the intended difficulty for specific boss mobs.

By properly configuring the Blacklist, you can ensure that your game balance remains intact while still allowing for customizations based on your gameplay preferences.

Custom Dimensions

To add support for custom dimensions from other mods, edit custom_dims.toml: 

                                [Custom Dimensions Config]
                                dimensions_settings = ["005/010/twilightforest:twilight_forest", "020/050/the_bumblezone:the_bumblezone"]

In this configuration:

  • dimensions_settings specifies the min and max levels for spawning in custom dimensions. Each entry follows the format (min/max/dimensionName), ensuring levels are in three-digit format. For example, (005/010/twilightforest:twilight_forest) limits spawning levels for the Twilight Forest to levels 5 through 10.

Debugging Commands

To enhance your modding experience, a new command has been added:

/Am_debug_dim

This command allows you to quickly retrieve the ID of the dimension you are currently in. Using this command can be particularly useful for configuring custom dimensions or troubleshooting dimension-specific behaviors.

How to Use:

  1. Open the chat in Minecraft.
  2. Type /Am_debug_dim and hit Enter.
  3. The dimension ID will be displayed in the chat, allowing you to use it in your configuration files.

Utilizing this command helps ensure that your settings are accurate and optimized for the dimensions you are working with.
